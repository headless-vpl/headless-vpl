# コアコンセプト: 4 つの型

全ての VPL は 4 つの普遍的パターンに分解できます。

## 1. 描画の型

**レスポンシブ**: 引数や数字の増減に応じてブロックが自動的に伸縮する仕組み。

**レイアウト**: CSS の Flexbox のような要素配置。VPL の形状を決定する。

| API | 説明 |
|-----|------|
| `Container` | メインコンテナ要素 |
| `AutoLayout` | CSS Flexbox ライクな自動レイアウト |
| `SvgRenderer` | デバッグ用ワイヤーフレーム描画 |

## 2. 接続の型

**コネクター**: 接続線を繋ぐ元と先にあるポイント。

**接続線**: コネクター間を結ぶ線。

**親子関係**: コネクターと接続線が形成する構造的な関係。

| API | 説明 |
|-----|------|
| `Connector` | 入出力接続ポイント |
| `Edge` | コネクター間の接続線（4 種のパスアルゴリズム） |
| `SnapConnection` | スナップ接続管理 |

## 3. 移動の型

**DnD**: マウスによるブロック/ノードの移動。

**スナップ**: 近づけると吸い付くようにくっつく機能。

**グループ**: 親を移動すると子も連動して動く。

| API | 説明 |
|-----|------|
| `InteractionManager` | DnD・パン・マーキー・リサイズ・Edge 作成を統合管理 |
| `SnapConnection` | スナップ接続管理 |

## 4. 入力の型

文字列・数字・トグル・スライダー等の入力要素。

Headless VPL では、入力要素は開発者が自由にフレームワーク（React, Vue 等）で実装します。ライブラリは座標と状態を管理し、見た目は開発者に委ねます。

## 単方向データフロー

```
フロントエンド（React / Vue / vanilla）
  → マウス/キーボードイベント
    → Headless VPL コア（Workspace, Container, Edge）
      → EventBus 通知
        → SvgRenderer（デバッグ用ワイヤーフレーム）
        → 開発者の DOM コンポーネント（本番 UI）
```
